How it Works?

The Virgil Security Stack includes cryptographic libraries that operate in conjunction with a
global cloud-based key management infrastructure-as-a-service that allow developers to add
security to their products in hours instead of weeks or months – without having to become
security experts themselves.

Our cryptographic libraries include an open-sourced hybrid symmetric/asymmetric encryption
schema (ECIES) with cryptographic agility that delivers substantially stronger security.

Ephemeral key pair generation

Input parameters (see PolarSSL documentation):
  • Random number generation function – NIST SP 800-901: CTR_DRBG (Counter-mode block-cipher-based Deterministic Random Bit Generator).
    The underlying algorithm used is AES-256 in counter mode.
  • Random number generation function “Personalization String” parameter – module id string (see section 8.7.1 of NIST Special
    Publication 800-90A
  • Entropy function – on linux: /dev/urandom and on Windows: CryptGenRandom(), with high resolution timer (rdtsc) and HAVEGE.

KA

Key Agreement – function used for the generation of a shared secret by two parties. PolarSSL ECDH implementation is used.

KDF

Key Derivation function – mechanism produces a set of keys from keying material and some optional parameters. Self-implementation of KDF1
described in the ISO-18033-2 Clause 6.2.2 is used. Underlying hash function is SHA-512.

ENC

Symmetric encryption algorithm. PolarSSL implementation of AES256-CBC mode is used.
